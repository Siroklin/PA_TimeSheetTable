# Руководство пользователя — «График работы сотрудников»

Документ разделён на общую часть (доступна всем), функции обычного пользователя
(роль «Редактирование») и раздел администратора.

---

## 1. Вход в систему

При открытии приложения показывается стартовый экран с логотипом и названием.

1. Нажмите кнопку **«Войти»**.
2. Введите **логин** и **пароль**, нажмите **«Войти»**.

Стандартный администраторский доступ при первой установке: логин `admin`, пароль `admin`.
**Сразу после первого входа смените пароль администратора** (раздел [5.2](#52-управление-пользователями)).

Сессия действует, пока открыта вкладка браузера — после закрытия вкладки нужно войти снова.
Чтобы выйти вручную: **Сервис → Выйти**.

---

## 2. Главный экран (доступно всем пользователям)

После входа открывается таблица графика работы за текущий месяц для одного из доступных вам отделов.

### 2.1 Панель фильтров

| Фильтр | Назначение |
|---|---|
| Переключатель месяца/года (стрелки `‹` `›` и выпадающие списки) | Период, за который показывается график |
| **Отдел** | Какой отдел сейчас показан (доступны только те отделы, которые вам назначены) |
| **Должность** | Показать только сотрудников выбранной должности |
| **Смена** | Показать только тех, у кого есть смены «День» или «Ночь» в этом месяце |

Справа сверху — кнопка **«Скачать Excel»**: выгружает текущий отдел/месяц в файл .xlsx.
Кнопка **«Сервис»** открывает меню действий — состав пунктов зависит от вашей роли (см. раздел [4](#4-роли-и-права-доступа)).

### 2.2 Таблица графика

- Строки — сотрудники (ФИО и должность), столбцы — дни месяца.
- Обычно каждый день разбит на колонки **Д** (день) и **Н** (ночь). Если у отдела включена настройка
  **«Нет ночных смен»** (раздел [5.1](#51-управление-отделами)), колонка «Ночь» скрывается, и на день
  показывается только одна колонка.
- При наличии прав редактирования ([раздел 3](#3-функции-обычного-пользователя-роль-редактирование)) возле
  имени сотрудника видны три значка: **«Граф.»** (быстрое заполнение графика по шаблону, см. [3.2](#32-быстрое-заполнение-по-шаблону-граф)),
  **✎** (изменить код/ФИО/должность сотрудника) и **×** (очистить график сотрудника за **текущий отображаемый месяц** — сам сотрудник в системе остаётся, удаляются только записи графика и шаблон за этот период).
  При роли «Просмотр» этих значков нет.
- Цвет ячейки соответствует статусу:

| Код | Статус | Цвет |
|---|---|---|
| *(без буквы, только число часов)* | Работает | зелёный |
| В | Выходной | жёлтый |
| О | Отпуск | синий |
| Б | Больничный | розовый |
| ДО | Отпуск за свой счёт | оранжевый (светлый) |
| П | Прогул | красный |
| К | Командировка | голубой |
| Ф | ФМС | фиолетовый |
| У | Увольнение | красный (яркий) |
| Д | Доп. смена | золотой |
| С | Сверхурочные | оранжевый |
| *(пусто)* | Не указано | серый |

Полная легенда с цветами также показана под таблицей графика на главном экране.

- Красная точка в ячейке — есть комментарий (виден при наведении или в окне редактирования ячейки).

### 2.3 Статистика по сотрудникам

**Сервис → Статистика по сотрудникам** — доступно **всем** пользователям, включая роль «Просмотр».

Показывает по каждому сотруднику текущего отдела/месяца:

| Колонка | Как считается |
|---|---|
| Смен (день) / Смен (ночь) / Смен всего | Количество отработанных смен (только явные рабочие часы, без учёта отпусков/больничных/командировок) |
| **Нормочасы** | Строго по **базовому графику сотрудника** (2×2, 5-0, 6-1, ДНОВ), внесённому через «Граф.» — сумма часов по паттерну за каждый день месяца. **Не зависит** от того, что реально стоит в ячейках. Если график на этот месяц не задан (не внесён через «Граф.» и не унаследован при копировании месяца) — показывается «—» |
| **Факт. часов** | Реальные отработанные/оплаченные часы по отметкам в ячейках — см. таблицу ниже, какие коды туда входят |
| Отклонение | Факт − Норма. Если норма неизвестна («—») — тоже «—» |

Какие коды статусов учитываются в «Факт. часов» (эта таблица также доступна прямо в форме статистики — раскрывающийся блок «Как коды статусов считаются в факте»):

| Код | Статус | В факте |
|---|---|---|
| *(число часов)* | Обычная смена | да |
| В | Выходной | нет |
| О | Отпуск | да |
| Б | Больничный | нет |
| ДО | Отпуск за свой счёт | нет |
| П | Прогул | нет |
| К | Командировка | да |
| Д | Доп. смена | да |
| С | Сверхурочные | да |
| Ф | ФМС | нет |
| У | Увольнение | нет |

## 3. Функции обычного пользователя (роль «Редактирование»)

Пункты этого раздела видны в меню «Сервис», только если ваша роль — «Редактирование» (или вы администратор).
Роль «Просмотр» их не видит и не может редактировать график (см. раздел [4](#4-роли-и-права-доступа)).

### 3.1 Изменить одну ячейку

1. Кликните по ячейке Д или Н нужного дня для нужного сотрудника.
2. Выберите статус (см. таблицу кодов в [2.2](#22-таблица-графика)).
3. Для всех статусов, кроме «Увольнение» и «Выходной», можно указать **часы работы** (0–11).
4. При необходимости укажите **«По (включительно)»** — дату окончания периода, если статус нужно
   проставить сразу на несколько дней подряд (например, отпуск на две недели). Для статуса «Увольнение»
   дата окончания автоматически ставится на последний день месяца.
5. При необходимости впишите комментарий.
6. Нажмите **«Сохранить»**.

Кнопка **«Удалить отсутствие»** в этом же окне отменяет статус и восстанавливает то значение, которое
должно было быть в этот день по базовому графику сотрудника (если график задан через «Граф.»).

### 3.2 Быстрое заполнение по шаблону («Граф.»)

Кнопка **«Граф.»** возле сотрудника открывает заполнение графика по шаблону на весь месяц:

| Шаблон | Описание |
|---|---|
| **2×2** | 2 рабочих дня / 2 выходных, скользящий цикл |
| **ДНОВ** | День → Ночь → Отсыпной → Выходной (4-дневный цикл, занимает обе колонки — Д и Н) |
| **5-0** | Пн–Пт работа, Сб–Вс выходной (в обиходе — «5/2») |
| **6-1** | Пн–Сб работа, Вс выходной |

Для **2×2** и **ДНОВ** нужно указать дату начала цикла (первый рабочий день) — это позволяет циклу
«продолжаться» корректно при копировании на следующие месяцы (см. [3.3](#33-копировать-расписание)).
Для **2×2**, **5-0** и **6-1** дополнительно выбирается смена сотрудника — дневная или ночная.

Если у отдела включена настройка **«Нет ночных смен»** — выбор смены и паттерн **ДНОВ** (требующий ночной
части) скрыты, заполнение идёт только по дневной колонке.

Внизу окна — предпросмотр первых 14 дней. Нажмите **«Применить»**, чтобы перезаписать график сотрудника
на весь месяц по шаблону (старые значения за этот месяц будут стёрты).

**Этот шаблон — основа для нормочасов** в статистике ([2.3](#23-статистика-по-сотрудникам)), поэтому его
рекомендуется вносить каждому сотруднику при найме, даже если график потом ведётся вручную по ячейкам.

### 3.3 Копировать расписание

**Сервис → Копировать расписание** — копирует график **всех сотрудников текущего отдела** из
отображаемого сейчас периода в выбранный целевой месяц/год.

- Если у сотрудника график был внесён через шаблон («Граф.»), при копировании скользящие циклы (2×2, ДНОВ)
  **продолжаются без разрыва** (а не начинаются заново), и сам шаблон (паттерн) переносится вместе с графиком.
- Если шаблон не использовался — данные копируются как есть, день-в-день, с попыткой подставить обычную
  рабочую смену вместо реальных отсутствий (отпуск/больничный/командировка и т.п. не переносятся, чтобы
  не растягивать разовое отсутствие на новый месяц).
- Уволенные сотрудники (статус «Увольнение» в исходном месяце) **не копируются** в целевой месяц и
  пропадают из списка отдела, пока не будут возвращены заново (см. [3.4](#34-сотрудники-и-справочники)).
- Существующий график в целевом месяце при копировании **будет перезаписан**.

### 3.4 Сотрудники и справочники

#### Добавить одного сотрудника

**Сервис → Добавить сотрудника** → код, ФИО, должность (из справочника должностей этого отдела), email
(необязательно) → **«Добавить»**.

**Возврат уволенного сотрудника.** Если сотрудник был уволен (статус «У») и через какое-то время
возвращается — просто добавьте его заново с **тем же табельным кодом** через эту форму. Система узнает,
что запись с таким кодом уже существует, но неактивна в текущем месяце, обновит его ФИО/должность и
вернёт в график этого месяца (вся история за прошлые периоды сохраняется). Если у сотрудника с этим кодом
уже есть график за текущий месяц — форма покажет ошибку «уже существует» (это защита от случайного дубля).

#### Изменить данные сотрудника

Значок **✎** в строке сотрудника — изменить код, ФИО, должность или email. На график это не влияет.

#### Загрузить список сотрудников (Excel/CSV)

**Сервис → Загрузить сотрудников**.

- Поддерживаются файлы **.xlsx/.xls** и **.csv/.txt**, либо можно просто вставить текст.
- Формат — 4 обязательные колонки и 1 необязательная: **код, отдел, ФИО, должность, email**. Для CSV/текста
  — через запятую, без заголовка (email можно оставить пустым):
  ```
  001,Цех №1,Иванов Иван Иванович,Оператор,ivanov@example.com
  002,Склад,Петрова Мария Сергеевна,Кладовщик,
  ```
- Для Excel-файлов первая строка может быть заголовком (`код`/`отдел`/`фио`/`должность`/`email` или
  английские `code`/`department`/`name`/`position`/`email` — распознаются автоматически); если заголовка
  нет, колонки читаются по порядку.
- Отдел в каждой строке должен совпадать с одним из существующих названий отделов.
- Нажмите **«Проверить»** — система покажет ошибки или таблицу предпросмотра по **текущему** выбранному
  файлу/тексту (если сначала выбрали не тот файл, а потом выбрали другой — превью и проверка теперь всегда
  соответствуют последнему выбранному файлу).
- Нажмите **«Загрузить»**: новые сотрудники добавляются; сотрудники, у которых код+отдел совпадает с уже
  существующим **активным** (есть график в этом месяце) — пропускаются как дубли; а вот ранее уволенные/
  неактивные в этом месяце сотрудники с тем же кодом — реактивируются (см. выше), а не пропускаются.

#### Управление должностями

**Сервис → Управление должностями** — справочник должностей конкретного отдела (используется в
выпадающем списке при добавлении сотрудника и в фильтре «Должность»).

- Переключить отдел можно прямо в этом окне.
- Добавить новую должность — введите название и нажмите **«Добавить»**.
- Удалить должность — кнопка **«×»** рядом с ней.

## 4. Роли и права доступа

Каждому пользователю (кроме администраторов) назначается:
- **список отделов**, которые он видит — в фильтре «Отдел» доступны только они;
- **роль**:

| Роль | Что доступно |
|---|---|
| **Редактирование** | Полный доступ в рамках своих отделов: просмотр, редактирование графика, добавление/изменение/возврат сотрудников, управление должностями, копирование расписания, скачивание Excel, статистика |
| **Просмотр** | Просмотр графика по своим отделам, скачивание Excel и **статистика по сотрудникам**. Пункты редактирования в меню «Сервис» не отображаются, клик по ячейке графика ничего не открывает |

**Администратор** не ограничен ролью и отделами — видит и может редактировать всё, плюс имеет доступ к
управлению отделами, пользователями и очистке расписания (раздел 5 ниже).

## 5. Раздел администратора

Пункты ниже видны только пользователям с правами администратора.

### 5.1 Управление отделами

**Сервис → Управление отделами**:
- Список всех отделов в системе.
- Чекбокс **«Нет ночных смен»** у каждого отдела — если включить, таблица графика для этого отдела
  показывает только дневную колонку (без «Н»), а заполнение по шаблону («Граф.») скрывает выбор смены и
  паттерн ДНОВ. Полезно для отделов, где ночных смен в принципе не бывает (например, склад, офис).
  Переключается сразу, без подтверждения.
- Добавить отдел — ввести название, при необходимости отметить «Нет ночных смен», нажать **«Добавить»**.
- Удалить отдел — кнопка **«×»**. Появится подтверждение: *«Вы точно хотите удалить отдел "X"? Все данные
  по отделу будут удалены.»* При подтверждении безвозвратно удаляются: все сотрудники отдела, их график,
  шаблоны заполнения, должности отдела и доступ пользователей к этому отделу.

### 5.2 Управление пользователями

**Сервис → Управление пользователями**:
- Список пользователей с указанием логина, имени, роли и доступных отделов.
- **Добавить пользователя**: имя, почта, логин, пароль, флажок «Администратор», роль
  («Редактирование»/«Просмотр»), отметить чекбоксами доступные отделы.
- **Редактировать** (значок ✎): те же поля; если оставить пароль пустым — он не меняется.
- **Удалить** (значок ×): нельзя удалить самого себя, чтобы не потерять доступ.

Если включён флажок «Администратор» — роль и список отделов не имеют значения, у пользователя полный
доступ ко всему.

### 5.3 Очистить расписание

**Сервис → Очистить расписание** (пункт выделен красным) — безвозвратно удаляет все записи графика и
шаблоны сотрудников выбранного отдела за один месяц **или сразу за весь год** (чекбокс «Очистить за весь
год»). Требует явного подтверждения чекбоксом с названием отдела и периода перед тем, как кнопка
«Очистить» станет активна. Сотрудники при этом из системы не удаляются — очищается только график.

## 6. Частые вопросы

**Не вижу нужный отдел в фильтре.**
Значит он вам не назначен — обратитесь к администратору, чтобы он добавил отдел в список доступных в
настройках вашего пользователя.

**Не вижу кнопок добавления сотрудников/«Граф.»/пункты в меню «Сервис», кроме статистики.**
У вашей учётной записи роль «Просмотр» — для редактирования обратитесь к администратору. Статистика и
скачивание Excel при этом доступны.

**В статистике у сотрудника нормочасы показывают «—».**
Значит на этот месяц у него не задан базовый график (шаблон) — ни через «Граф.», ни унаследован при
копировании месяца. Внесите шаблон через «Граф.» (см. [3.2](#32-быстрое-заполнение-по-шаблону-граф)).

**Сотрудник уволился, а через пару месяцев вернулся — как его добавить обратно?**
Через «Добавить сотрудника» с тем же табельным кодом, что и раньше — см.
[3.4](#34-сотрудники-и-справочники). Заново заводить его под другим кодом не нужно и не рекомендуется —
это разорвёт историю.

**Случайно перезаписал график при копировании или заполнении по шаблону.**
Действие необратимо откатить через интерфейс нельзя — данные за месяц нужно будет ввести заново. Будьте
внимательны при выборе целевого месяца перед копированием.

**Забыли пароль.**
Только администратор может сбросить пароль — через «Управление пользователями» → редактирование
пользователя → ввести новый пароль.
